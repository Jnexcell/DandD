# Modules — generated one-shots

This folder holds **ready-to-run one-shot modules** produced by the Session/Module Creator
(`../tools/session-creator.md`). Each file is a self-contained session you can run at the table without
flipping books — read-aloud text, NPCs, encounters **auto-scaled to your party** with **full stat blocks
inlined**, skill DCs, branching avenues, two endings, and a hook.

## How to make one

Say *"build me a one-shot"* (or *"use the session creator"*) and answer the intake questions
(party size & level, length, tone, setting, premise, must-haves, safety lines). Claude drafts the whole
module in one pass, saves it here as `modules/<slug>.md`, then refines on your notes. See
[`../tools/session-creator.md`](../tools/session-creator.md) for the full workflow.

## Anatomy of a module

Built from the structure in [`_TEMPLATE.md`](_TEMPLATE.md): a one-page overview (incl. the filled MMOS
worksheet) → how-to-run notes → cold open → map of play → scenes → the Showdown → two endings → hooks →
appendix.

## The modular convention (read this)

Every scene is tagged:

- **★ CORE** — always run. Only the **cold open** and the **Showdown** are core.
- **⬡ MODULAR** — skippable. Each modular scene carries a **Bypass** line telling you how to cut it for
  time while still handing the party anything they need to proceed.

This is the heart of the MMOS method: a short critical path (cold open → direct route → Showdown) plus
optional content you can drop on the fly to hit your time budget. Each module also includes a **cut-list**
(the order to drop scenes if you're running long).

## Scope

One-shots only for now, with monsters in the **CR 0–5** range (where the KB has full stat blocks).
Campaign mode, the Character Creation Helper (2b), and the live DM Assistant (2c) are separate builds —
but these module files are structured so 2c can later walk them scene by scene.
