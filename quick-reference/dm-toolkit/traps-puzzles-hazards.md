# DM Toolkit — Traps, Puzzles & Hazards

**Purpose:** drop a non-combat obstacle into a scene on demand — a trap to disarm, a puzzle to crack, a hazard to survive, or a skill-challenge to push through. The **trap/hazard rules and sample traps** already live in the verified DMG ch.5 file; this toolkit gives the fast build-scaffold + the things the DMG is thin on: **reusable puzzle patterns** and **skill-challenge framing**.

**Grounded in / pulls from:**
- `dungeon-masters-guide/05-adventure-environments.md` — trap rules, **Trap Save DC / Attack Bonus** + **Damage Severity by Level** tables, complex traps, and 11 fully statted sample traps (DMG p.120–123); dungeon hazards (DMG p.105).
- `quick-reference/difficulty-classes.md` — DC ladder for setting check DCs.
- `dungeon-masters-guide/08-running-the-game.md` — ability checks, group checks, and the social/exploration framing a skill challenge leans on.

## Trap in 5 lines (build one live)

1. **Trigger** — what sets it off? (pressure plate, trip wire, opening a lock/chest/door, weight on a tile, touching an object, speaking a word).
2. **Effect** — what happens? (damage, a condition, separation, alarm, a falling/closing/flooding hazard).
3. **Severity** — pick **Setback / Dangerous / Deadly** for the party's level, then read the **save DC or attack bonus** and the **damage dice** straight off the tables in `05-adventure-environments.md` (DMG p.121). Spine: Setback DC 10–11 / +3–5; Dangerous DC 12–15 / +6–8; Deadly DC 16–20 / +9–12. Damage (L1–4): Setback 1d10 · Dangerous 2d10 · Deadly 4d10.
4. **Detect** — passive Perception or an active check to notice it (Wis (Perception) to spot, Int (Investigation) to understand). Set the DC off the ladder.
5. **Disarm / bypass** — the *out*: Dex (thieves' tools) to disable, a spike to wedge a plate, *dispel magic* for a magic trap, or simply going around. **Always give at least one bypass** — a trap with no solution is just a damage tax.

> **Don't reinvent — reskin.** The 11 sample traps in `05-adventure-environments.md` (collapsing roof, falling net, fire-breathing statue, pits, poison darts/needle, rolling sphere, sphere of annihilation) are fully statted. Grab the closest one and change the dressing.
>
> **Make it a fight:** a **complex trap** rolls initiative and acts each round (a room flooding a level per turn, a blade-pendulum) — use it for a tense set-piece (DMG p.121–122).

**Telegraph traps.** Scorch marks, suspicious clean floor tiles, a skeleton clutching a lever, dart-holes in the wall. A fair trap is one the players had a *chance* to read.

## Hazards (environmental, not built) — DMG p.105

Work like traps but are part of the terrain: **brown mold** (cold, and fire makes it spread), **green slime** (acid, drops from ceilings), **yellow mold** (poison spores), **webs**, **frigid water**, **quicksand**, **razorvine**. Spotting needs no check unless hidden; *identifying* a benign-looking patch is an **Int (Nature)** check. Gauge lethality with the **Damage Severity by Level** table. Use them to make a corridor a *decision* ("the only dry path runs right past the slime").

## Puzzle design patterns (reusable templates)

The DMG doesn't give a puzzle system — here are patterns you can fill with any theme. **Always bake in a hint path and a fail-forward** so a stuck party isn't a dead session.

| Pattern | The core | Example dressing | The "out" / hint |
|---|---|---|---|
| **Riddle / password** | Answer a question or speak a phrase | A sphinx's riddle; a door that asks "what walks the dark unseen?" | Clues seeded earlier; Int (History/Arcana) or a lore NPC; 3 wrong tries → it just opens but wakes a guard |
| **Sequence / pattern** | Do steps in the right order | Light braziers in the order of the seasons; press constellation tiles in the sky-map's order | The order is written/painted somewhere in the dungeon; Investigation finds it |
| **Matching / sorting** | Pair or group things correctly | Place the right gem in the right god's statue; sort tomes by date | A plaque or mural shows the correct associations |
| **Hidden in plain sight** | The solution is *present* but disguised | The "decoration" is the key; the poem's first letters spell the word | Perception/Investigation; reading the thing aloud |
| **Physical manipulation** | Move/weigh/rotate to a state | Counterweight a scale; rotate rings so symbols align; redirect light with mirrors | Trial-and-error is allowed; checks speed it up, don't gate it |
| **Information gathering** | Assemble clues from multiple places | Three plaques each give one digit of a vault code | Each clue is its own mini-find; partial info still helps |
| **Sacrifice / cost** | Progress demands giving something up | A bridge that needs a drop of blood (HP); a door paid in gold or a memory | The cost is the puzzle; make it meaningful, not lethal-random |

**Puzzle rules of thumb:**
- **Solve it with the table, not the character sheet** — let players *think*, but offer a check (Int, or the relevant skill) as a hint/nudge so a low-Int party isn't locked out.
- **Three-clue rule:** for anything gating progress, plant **three** ways to reach the answer. Players miss clues.
- **Fail forward:** failure costs *time, resources, or noise* (a wandering monster, a sprung trap) — never a hard stop.
- **One puzzle per dungeon** is usually plenty for a one-shot; it's seasoning, not the meal.

## Skill challenges (structured non-combat obstacles)

When the obstacle is an *extended effort* — crossing a storm-lashed mountain, talking down a mob, chasing a thief across rooftops, performing a ritual under attack — run it as a **skill challenge** instead of a single roll.

**The frame:**
1. **State the goal and stakes** ("reach the pass before nightfall, or the blizzard hits").
2. **Set a target: X successes before Y failures.** Rule of thumb — **easy 4/3, standard 6/3, hard 8/3** (successes before 3 failures).
3. **Set the DC** off the ladder (`difficulty-classes.md`) — usually Medium (15), Hard (20) for risky steps.
4. **Each PC describes an approach**, then rolls the skill that fits the *fiction they proposed* (reward creativity; the same skill twice in a row gets harder or disallowed — no "everyone rolls Athletics").
5. **Narrate every roll**, success or fail — each one moves the story (ground gained, a setback, a complication).
6. **Resolve:** hit X successes → they make it (maybe scarred). Hit Y failures → they make it *the hard way* (lost time, hurt, a fight) — **fail forward**, don't dead-end.

> A skill challenge is also a great **avenue** (see `scenes-goals-and-avenues.md`) and pairs with **social interaction** (see `social-and-dialog.md`) when the "obstacle" is a person.

## 20-second checklist (use this live)

1. **What kind of obstacle?** trap (`05-adventure-environments.md`) / hazard / puzzle (patterns above) / skill challenge.
2. **Trap:** trigger + effect + severity band → DC & damage from the DMG table; **give a bypass**; telegraph it.
3. **Puzzle:** pick a pattern, dress it, plant **three** clue paths, decide the fail-forward cost.
4. **Skill challenge:** goal + X/3 successes + DCs; let players pick skills from the fiction; narrate each roll.
5. **Always:** make failure *interesting*, not a wall.
